> **Проект:** [sud.cvr.name](https://sud.cvr.name) — платформа для мониторинга судебных дел и проверки контрагентов
> **Период:** Май 2026

## 📋 Описание проекта

**Sud** — это fullstack-приложение (React 19 + Supabase + Vite) для юристов, предпринимателей и граждан, позволяющее:
- Отслеживать судебные дела в судах РФ
- Проверять контрагентов (компании и физических лиц)
- Получать уведомления о новых событиях
- Использовать AI-юриста (Gemini)

**Стек технологий:**
- Frontend: React 19, TypeScript, Tailwind CSS, Vite, Framer Motion
- Backend: Supabase (PostgreSQL, Edge Functions, Auth, Storage)
- AI: Google Gemini API
- Инфраструктура: Vercel (frontend), Supabase Cloud (backend + БД)

#### Frontend
- **`/counterparty/:inn`** — страница проверки контрагента
- **Дашборд** с разворачиваемыми секциями (аккордеон)
- **Риск-скоринг (6 факторов):**
  - Исполнительные производства (+10% за каждое, max 30%)
  - Банкротство (+50%)
  - Ликвидация (+40%)
  - Реорганизация (+20%)
  - Нулевой капитал (+15%)
  - Массовый адрес (+10%)
- Прогресс-бар риска (0-100%) с цветовой индикацией:
  - 🟢 <20% — "Низкий риск"
  - 🟡 20-50% — "Средний риск"
  - 🔴 >50% — "Высокий риск"

#### Интеграция
- Добавлена категория "Контрагенты" на главной странице
- Быстрая ссылка в разделе "Мониторинг"

  - Оппоненты: ТОП-5 компаний с кем судятся
  - Судьи: ТОП-5 судей + % удовлетворённых исков
- **Цветовая индикация статуса:** зелёный (удовлетворено), красный (отказ), серый (в процессе)

## 💡 Ключевые решения

1. **Fallback-стратегия:** Если внешний API недоступен — показываем ошибку, но не ломаем UI
2. **Параллельные запросы:** `Promise.allSettled()` для всех проверок контрагента
3. **Edge Functions:** Вся логика работы с внешними API вынесена в Supabase Edge Functions (Deno)
4. **Риск-скоринг:** Автоматический расчёт на основе реальных данных, а не случайных чисел
5. **Модульность:** Каждый этап — независимый модуль, можно включать/выключать
