# Design Spec: Дополнительные API и улучшение мониторинга

> **Дата:** 2026-05-18
> **Проект:** sud.cvr.name (Casebook alternative)
> **Статус:** Ожидает одобрения

---

## Обзор

Расширение функционала проверки контрагентов и мониторинга. Внедрение 3 новых бесплатных API и Web Push уведомлений в систему автомониторинга.

---

## Модули

### Модуль 1: Росстат (Бухгалтерская отчётность)

**API:** `bo.nalog.ru` (ФНС, бесплатно, официально)

**Функционал:**
1. Поиск организации по ИНН
2. Получение бухгалтерских данных:
   - Баланс (активы, обязательства, капитал)
   - Отчёт о прибылях и убытках (выручка, прибыль, расходы)
   - Кредиторская/дебиторская задолженность
   - Динамика по годам (2015-2025)
3. Расчёт финансовых коэффициентов:
   - Рентабельность продаж
   - Коэффициент текущей ликвидности
   - Доля заёмных средств

**UI:**
- Новая секция "Финансы" на дашборде контрагента (CounterpartyDashboard)
- Таблица показателей + график динамики (Recharts или Chart.js)
- Индикатор "Высокая доля задолженности" если кредиторка > 70% активов

**Edge Function:** `check-rosstat` (новая)

**Тесты:**
- Unit: расчёт коэффициентов
- Component: отображение финансовых данных

### Модуль 2: Аффилированность (Граф связей)

**API:** ЕГРЮЛ (уже реализован `check-egrul`) + `check-egrul` доп. запросы

**Функционал:**
1. Извлечение учредителей из ЕГРЮЛ выписки
2. Для каждого учредителя (если указан ИНН физлица):
   - Поиск компаний, где он учредитель (через тот же `check-egrul`)
   - Ограничение: 1 уровень (учредители данной компании + их компании)
3. Визуализация графа связей:
   - Центральный узел: проверяемая компания
   - Соседние узлы: учредители (физлица)
   - Узлы второго уровня: другие компании этих учредителей
   - Библиотека: `react-force-graph-2d` (лёгкая, open-source)

**UI:**
- Новая секция "Связи" на дашборде контрагента
- Интерактивный граф (перетаскивание, зум, hover для деталей)
- Список "Связанные компании" с риск-скорингом каждой

**Ограничения:**
- Обратный поиск по ФИО без ИНН — невозможен бесплатно
- Глубина: максимум 2 колена (компания → учредитель → его компании)

**Edge Function:** Реализуется на frontend через повторный вызов `check-egrul`

**Тесты:**
- Unit: парсинг учредителей из ЕГРЮЛ
- Component: отображение графа

### Модуль 3: ГАС Правосудие (Суды общей юрисдикции)

**API:** Парсинг `sudrf.ru` (каталог судебных решений) + `https://bsr.sudrf.ru` (банк данных судебных решений)

**Функционал:**
1. Поиск судебных решений по:
   - ИНН компании / физлица
   - ФИО стороны
   - Номеру дела
2. Данные по делу:
   - Номер, дата, суд
   - Стороны (истец/ответчик)
   - Текст решения (ссылка на PDF)
   - Судья
3. Статистика:
   - Количество дел по категориям
   - Динамика по годам

**UI:**
- Новая страница `/civil-cases` (аналог `/arbitration` для общей юрисдикции)
- Поиск по номеру дела / стороне / судье
- Карточки дел с ссылками на решения
- Дашборд статистики (при поиске по компании)

**Edge Function:** `search-civil-cases` (новая)

**Тесты:**
- Unit: парсинг результатов поиска
- Component: отображение результатов

### Модуль 4: Web Push уведомления (Мониторинг)

**Технология:** Web Push API + Service Worker + VAPID

**Функционал:**
1. **Подписка:**
   - Пользователь нажимает "Включить уведомления" в настройках
   - Браузер запрашивает разрешение
   - Сохранение push-подписки в таблице `push_subscriptions`

2. **Отправка:**
   - Edge Function `monitor-companies` при обнаружении изменений:
     - Проверяет `push_subscriptions` для user_id
     - Отправляет push-уведомление через Web Push API
   - Сообщение: "⚠️ Изменения в [Название компании]: новое ИП / смена директора / банкротство"

3. **Управление:**
   - В настройках профиля: включить/выключить push
   - Удаление подписки при отписке

**UI:**
- Кнопка "Включить push-уведомления" на странице `/monitoring`
- Badge с количеством непрочитанных событий

**Таблица:**
```sql
CREATE TABLE push_subscriptions (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id uuid REFERENCES auth.users,
    endpoint text NOT NULL,
    p256dh text NOT NULL,
    auth text NOT NULL,
    created_at timestamptz DEFAULT now()
);
```

**Edge Functions:**
- Обновление `monitor-companies`: добавить отправку push
- Новая `send-push-notification` (вызов Web Push API)

**Тесты:**
- Unit: мокирование Web Push API
- Component: кнопка подписки/отписки

## Порядок реализации

1. **Росстат** — проще всего, чёткий API
2. **Push-уведомления** — инфраструктурная задача, используется остальными
3. **Аффилированность** — зависит от ЕГРЮЛ, визуализация
4. **ГАС Правосудие** — парсинг, может быть нестабильным

---

## Риски и ограничения

| Риск | Митигация |
|---|---|
| `bo.nalog.ru` API изменится | Fallback: показывать ошибку, но не ломать UI |
| `bsr.sudrf.ru` блокирует парсинг | Кэширование, retry с backoff, заглушка "Сервис временно недоступен" |
| Обратный поиск учредителей невозможен | Показывать только связи известные из ЕГРЮЛ, без второго уровня если нет ИНН |
| Push-уведомления не поддерживаются Safari | Graceful degradation, показывать email как fallback |
| VAPID ключи нужно сгенерировать | Хранить в Supabase Secrets |

## Технологии

- **Граф:** `react-force-graph-2d` (D3-based, лёгкий)
- **Графики финансов:** `recharts` (уже в проекте через shadcn)
- **Web Push:** `web-push` (npm package) или нативный Web Push API
- **Парсинг ГАС:** Cheerio или нативный DOMParser в Deno
