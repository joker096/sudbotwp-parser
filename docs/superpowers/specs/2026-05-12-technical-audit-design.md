# Технический аудит проекта sud.cvr.name

## Цель

Провести обзор кодовой базы, выявить проблемные места и быстрые улучшения (quick wins).

## Области аудита

### 1. TypeScript — Улучшение типизации

**Проблема:**
- 28 мест с `: any` в определениях
- 261 место с `as any` в коде

**Решение:**
- Создать типы для часто используемых объектов (Case, Event, Appeal, Profile)
- Заменить `any` на конкретные типы в `supabase.ts`
- В тестах оставить `as any` — это нормально для mock-ов

**Приоритет:** Средний

### 2. Большие файлы — Рефакторинг

**Проблема:** 5 файлов >1000 строк
- Profile.tsx — 2281 строка
- HtmlEditor.tsx — 2218 строк
- supabase.ts — 2792 строки
- AdminSettings.tsx — 1435 строк
- CaseCard.tsx — 1276 строк

**Решение:**
- Разделить Profile.tsx на логические блоки: ProfileHeader, ProfileTabs, ProfileStats
- Разделить AdminSettings.tsx на меньшие компоненты (по секциям настроек)
- CaseCard.tsx — вынести в подкомпоненты (CaseCardHeader, CaseCardFooter)

**Приоритет:** Высокий

### 3. Edge Functions — Типизация ошибок

**Проблема:** Все 16 функций используют `catch (error)` без типа

**Решение:**
- Добавить тип `Error` для catch параметров: `catch (error: Error)`
- Добавить логирование stack trace: `console.error(error.stack)`

**Приоритет:** Низкий

### 4. React Hooks — Проверка зависимостей

**Проблема:** 5 хуков с useEffect

**Решение:**
- Проверить все useEffect на наличие cleanup функций
- Проверить зависимости в массивах (exhaustive-deps)
- useAuth уже имеет защиту от Strict Mode

**Приоритет:** Средний

### 5. Безопасность

**Проблема:** HtmlEditor использует dangerouslySetInnerHTML

**Решение:**
- Проверить XSS protection в HtmlEditor (используется DOMPurify)
- Добавить SafeHtml компонент-обёртку
- SafeLink для внешних ссылок уже реализован

**Приоритет:** Высокий

## Результат

- Список найденных проблем с приоритетами
- Быстрые исправления (1-2 часа): типы в supabase.ts, XSS проверка
- Средние улучшения (1-2 дня): рефакторинг Profile.tsx, AdminSettings.tsx
