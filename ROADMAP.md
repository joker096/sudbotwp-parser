## 🎯 Этапы внедрения (по порядку)

### ✅ Этап 1: Проверка контрагентов (Компании/ИП)
**Приоритет:** Высокий | **Срок:** 2-3 недели | **Статус:** Завершено

**Что внедряем:**
1. **ЕГРЮЛ/ЕГРИП через `egrul.nalog.ru`** (ФНС)
   - Полная выписка: директор, учредители, ОКВЭД, адрес, статус, уставной капитал
   - История изменений (дата + что изменилось)
   - API: парсинг или `https://egrul.nalog.ru` (бесплатно, без лимитов)

2. **ФССП проверка** (`api-ip.fssprus.ru/api/v1`)
   - Исполнительные производства по ИНН
   - Сумма долга, статус, пристав
   - Лимит: 500 запросов/сутки (бесплатно)

3. **ЕФРСБ (банкротство)** (`bankrot.fedresurs.ru/backend/`)
   - Банкротство компании/физлица
   - Реестр требований кредиторов
   - Сообщения арбитражного управляющего

4. **Росстат (бухгалтерская отчётность)** (`bo.nalog.ru`)
   - Выручка, прибыль, активы, кредиторская/дебиторская задолженность
   - График динамики финансов

5. **Арбитражные дела (КАД)** по компании (`kad.arbitr.ru`)
   - Поиск дел по ИНН/названию
   - Статистика: истец/ответчик/третье лицо
   - Суммы исков

6. **Аффилированность (граф связей)**
   - По учредителям из ЕГРЮЛ: найти все компании этого учредителя
   - Визуализация связей (D3.js или Cytoscape.js)
   - Глубина: 1-2 колена (бесплатно через ЕГРЮЛ)

7. **Риск-скоринг (35 факторов)**
   - Автоматический расчёт на основе собранных данных:
     - Исполнительные производства → +30% риска
     - Банкротство → +50% риска
     - Частые смены директора/адреса → +20%
     - Массовый адрес → +15%
     - Нулевая выручка 3 года → +25%
     - Много арбитражей как ответчик → +20%
     - Ликвидация/реорганизация → +40%
   - Итоговый балл 0-100% + прогноз ("Надёжный", "Средний риск", "Высокий риск")

**Результат:** Страница `/counterparty` или улучшение `/monitoring` с полным дашбордом компании как в Casebook.

### ✅ Этап 2: Проверка физических лиц
**Приоритет:** Средний | **Срок:** 1 неделя | **Статус:** Завершено

**Что внедряем:**
1. **ФССП по физлицу** — тот же API, что и для компаний (ИНН 12 цифр)
2. **ЕФРСБ по физлицу** — тот же API (проверка банкротства физлиц)
3. **Росфинмониторинг** (`fedsfm.ru`)
   - Проверка в списке террористов/экстремистов
   - Парсинг XML/JSON списка
4. **Недействительные паспорта** (API МВД или парсинг `паспорт.рф`)
5. **ГИБДД штрафы** (парсинг `гибдд.рф` — сложно из-за капчи, можно отложить)

**Результат:** Новая страница `/person-check` или расширение `/taxpayer` до полной проверки физлица.

---

### ✅ Этап 3: Арбитражная статистика
**Приоритет:** Средний | **Срок:** 2 недели | **Статус:** Завершено

**Что внедряем:**
1. **Поиск по КАД** (`kad.arbitr.ru`)
   - По номеру дела, сторонам, судье, суду, ИНН
   - Карточка дела: стороны, судья, даты, решения

2. **Статистика судьи**
   - Собрать все дела судьи из КАД
   - % удовлетворённых исков, % отказов, % возвратов
   - По категориям споров

3. **Анализ оппонентов**
   - С кем чаще всего судится компания
   - Суммы исков по каждому оппоненту
   - Диаграмма "истец/ответчик"

4. **Исковая нагрузка**
   - Динамика по годам (2020-2025)
   - Сравнение со средним по отрасли (ОКВЭД)
   - Графики: количество дел, суммы, исходы

5. **Предсказание исхода дела**
   - Использовать существующий AI (Gemini) + данные из статистики судьи
   - "Вероятность удовлетворения иска: 75% (судья Иванов в 80% случаев удовлетворяет иски в этой категории)"

**Результат:** Новый раздел `/arbitration` с поиском КАД и аналитикой.

---

### ✅ Этап 4: Мониторинг компаний (автоматический)
**Приоритет:** Средний | **Срок:** 1-2 недели | **Статус:** Завершено

**Что внедряем:**
1. **Подписка на компанию по ИНН**
   - Сохранение в таблицу `monitored_companies`
   - Cron-задача (pg_cron в Supabase): раз в сутки проверять:
     - ЕГРЮЛ (изменения статуса, директора, адреса)
     - ФССП (новые ИП)
     - ЕФРСБ (новые банкротства)
     - КАД (новые дела)

2. **Лента событий**
   - Таблица `company_events`: "Изменился директор", "Новое ИП на 500 000₽", "Новое дело А40-12345/2026"
   - Фильтры по типу событий

3. **Email-дайджест**
   - Раз в неделю: сводка изменений по всем отслеживаемым компаниям
   - Использовать существующую функцию `send-report-email`

4. **Telegram-уведомления**
   - Мгновенные уведомления о важных событиях (банкротство, новое ИП)
   - Использовать существующий `send-telegram-notification`

**Результат:** Раздел `/monitoring` → вкладка "Мои контрагенты" с автомониторингом.

### ✅ Этап 5: Интеграции
**Приоритет:** Низкий | **Срок:** 1 неделя | **Статус:** Завершено

**Что внедряем:**
1. **Мой Арбитр API** (бесплатный тир)
   - Проверка статуса дела в электронном виде
   - Список документов дела (если API позволяет)

2. **Экспорт PDF-отчёта**
   - По кнопке "Скачать отчёт" на странице проверки контрагента
   - Генерация PDF через Puppeteer/Playwright или библиотеку (jsPDF/html2pdf)

3. **Автозаполнение шаблонов**
   - На основе данных из ЕГРЮЛ автозаполнить:
     - Исковое заявление (истец/ответчик реквизиты)
     - Ходатайство (реквизиты суда из таблицы `courts`)

**Результат:** Улучшение существующего функционала.

## 🔧 Техническая архитектура

### Таблицы Supabase (уже созданы или требуют создания):

```sql
-- Отслеживаемые компании
CREATE TABLE monitored_companies (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES auth.users,
  inn text NOT NULL,
  name text,
  ogrn text,
  risk_score integer DEFAULT 0,
  status text DEFAULT 'active',
  last_check_at timestamptz,
  created_at timestamptz DEFAULT now()
);

-- События по компаниям (лента)
CREATE TABLE company_events (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  company_id uuid REFERENCES monitored_companies,
  event_type text, -- 'egrul_change', 'fssp_new', 'bankruptcy', 'new_case'
  event_data jsonb,
  created_at timestamptz DEFAULT now()
);

-- Проверки контрагентов (кэш)
CREATE TABLE counterparty_checks (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  inn text NOT NULL,
  name text,
  egrul_data jsonb,
  fssp_data jsonb,
  efrsb_data jsonb,
  rosstat_data jsonb,
  kad_data jsonb,
  risk_factors jsonb,
  checked_at timestamptz DEFAULT now()
);
```

## 💰 Что остаётся платным (как в Casebook)

| Функция | Почему платно | Альтернатива бесплатно |
|---|---|---|
| Мой Арбитр (подача документов) | Требует ЭП + API Мой Арбитр Pro | Ручная подготовка + шаблоны |
| Исторические выписки ЕГРЮЛ на любую дату | ФНС не хранит историю бесплатно | Только текущая выписка |
| Аффилированность 30 колен | Нужна огромная база связей | 1-2 колена через ЕГРЮЛ |
| Проверка паспорта в реальном времени | API МВД платный | Периодически обновляемый список |
| Мониторинг каждый час | Нагрузка на сервер | Раз в сутки (cron) |
| API доступ для внешних систем | Инфраструктура | Только внутренний UI |

## ✅ Этап 6: Росстат (Бухгалтерская отчётность)
**Приоритет:** Высокий | **Статус:** Завершено

**Что внедряем:**
1. **Росстат (`bo.nalog.ru`)** — бухгалтерская отчётность
   - Баланс: активы, обязательства, капитал

## ✅ Этап 7: Аффилированность (Граф связей)
**Приоритет:** Средний | **Статус:** Завершено

**Что внедряем:**
1. **Парсинг учредителей из ЕГРЮЛ**
   - Извлечение ФИО и долей из выписки
   - Список учредителей с долями
2. **Граф связей**
   - Центральный узел: проверяемая компания

**Ограничение:** Обратный поиск по ФИО без ИНН невозможен бесплатно.

**Результат:** Секция "Связи" на дашборде контрагента.

**Приоритет:** Средний | **Статус:** Завершено

## ✅ Этап 10: Производительность и UX
**Приоритет:** Высокий | **Статус:** Завершено

**Что внедряем:**
1. **Кэширование проверок**
   - Таблица `counterparty_checks` (TTL 24 часа)

   - `OfflineBanner` — баннер при отсутствии сети
   - Background Sync в Service Worker для отложенных проверок

**Результат:** Ускорение повторных проверок, экспорт данных, offline-режим.
